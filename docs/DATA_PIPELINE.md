# Data Pipeline — K-Tactics Lab 2026

이 문서는 K-Tactics Lab의 **데이터 수집 → 정제 → 변환 → 검증** 전 과정을 재현 가능한 수준으로 기록한다. 요약본은 [`AI_ENGINEERING.md`](AI_ENGINEERING.md) §3에 있고, 여기서는 공식·근거·검증 결과를 전부 펼친다. 이 파이프라인의 산출물(`data/squad_stats_2026.js`)은 게임의 팀 밸런스 HUD, 몬테카를로 λ 유도, 결과 카드의 XAI 요인 분해까지 모든 "실제 연산"의 단일 데이터 소스다.

```
FBref 원자료 (Standard/Shooting/Misc, 3개 표)
   │  ① 수집: 실제 출전 20인 스코프 확정
   ▼
scripts/parse_stats.py 의 PLAYERS 배열 (원자료 + 큐레이션 필드)
   │  ② 정제: per-90 정규화, 소표본 태깅, 수동 보정 명시
   ▼
4대 능력치 변환 공식 (포지션 프라이어 + 표본 가중 + 클램프)
   │  ③ 변환: attack / defense / midfield / stamina (30~96)
   ▼
data/squad_stats_2026.js (정적 자산, 런타임 $0)
   │  ④ 검증: 재현성 diff-zero, 분포·서열·커버리지 체크
   ▼
app.js 소비: 팀 밸런스 HUD → λ 유도 → 1,000회 몬테카를로 → XAI 분해
```

---

## 1. 수집 (Collection)

- **출처**: FBref, Korea Republic 팀 페이지, World Cup 2026 대회 필터. 사용한 표는 **Standard**(출전/득점/도움), **Shooting**(Sh/90, SoT/90), **Misc**(Int, TklW, Fls, Fld, Crs) 3개.
- **스코프 규칙**: 소집 명단이 아니라 **실제 출전 기록(MP > 0)이 있는 20인만** 포함한다. 미출전 선수를 섞으면 "표본 없는 능력치"를 지어내야 하므로 제외했다.
- **커버리지 체크섬**: 20인의 출전 시간 합 = **2,970분 = 3경기 × 11명 × 90분**. 팀이 소화한 전체 선수-분과 정확히 일치하므로, 이 스코프가 대회 출전 기록을 100% 커버함을 산술적으로 확인할 수 있다.

## 2. 정제와 스키마 (Cleaning & Schema)

원자료는 `scripts/parse_stats.py`의 `PLAYERS` 배열에 선수당 1행으로 입력한다.

| 필드 | 뜻 | 성격 |
|---|---|---|
| `n`, `p`, `a` | 이름, FBref 포지션(복합 포함: FW/MF 등), 나이(`년-일`) | 원자료 |
| `mp`, `st`, `min`, `g`, `as` | 경기수, 선발, 출전분, 골, 도움 | 원자료 |
| `s90` | 90분 환산 출전량 (= min/90) | 원자료 파생 |
| `sh`, `sot` | Sh/90, SoT/90 | 원자료 |
| `int`, `tkl`, `fls`, `fld`, `crs` | 인터셉트, 태클 성공, 파울, 피파울, 크로스 (대회 총계) | 원자료 |
| `save_pct` | GK 세이브율 | 원자료 (GK만) |
| `rtg`, `sh2`, `cmp` | 카드 종합 레이팅, 슈팅, 침착성(PK용) | **큐레이션** |
| `note`, `override_def` | 표본 주석, 수동 보정 | 큐레이션 (명시적) |

정직성 원칙 두 가지:

1. **파생과 큐레이션을 구분한다.** 4대 능력치(공/수/중/체력)는 아래 공식으로 **계산**되지만, `rating`/`shooting`/`composure`는 FBref 수치와 통념을 참고한 **편집 값**이다. 이 구분을 숨기지 않는다 (기획서·심사 답변에서도 동일).
2. **표시하지 않는 데이터를 지표인 척 두지 않는다.** 초기의 `npxG` 등 미측정 플레이스홀더는 전부 제거했다 ([`AI_ENGINEERING.md`](AI_ENGINEERING.md) §3).

## 3. 변환 공식 (Transformation)

모든 능력치는 **포지션 프라이어(base) + per-90 실적 가산 + 표본 가중 + 클램프** 구조다. 3경기 대회 데이터는 표본이 작으므로, 출전량이 적을수록 실적 항의 영향을 줄이는 가중치 `w`가 핵심 설계다.

```
attack   = ATK_BASE[pos] + min(10, Sh/90 × 2.5)·w + min(6, SoT/90 × 4)·w + 골×5 + 도움×3        (w = min(1, s90/2.0))
defense  = DEF_BASE[pos] + min(12, Int/s90 × 5)·w₂ + min(8, TklW/s90 × 4)·w₂ − 파울과다 페널티·w₂  (w₂ = min(1, s90/1.5))
  └ GK 분기: 82 + (Save% − 70) × 0.3   (세이브율 기반 별도 공식, 70~96)
midfield = MID_BASE[pos] + min(12, Crs/s90 × 3)·w + 도움×4 + min(5, 피파울) + 골×2
stamina  = 85 − max(0, 나이−27)×0.6 + min(3, max(0, 25−나이)×0.4) + 포지션 보정   (나이 감쇠 모델)
```

- **포지션 프라이어**: FW의 공격 base 72 vs GK 20처럼, 포지션이 이미 담고 있는 역할 정보를 초기값으로 쓴다. 복합 포지션(FW/MF 등)은 별도 행으로 세분화했다.
- **per-90 정규화**: 총계 스탯(Int, TklW, Crs)은 `s90`으로 나눠 출전 시간 차이를 제거한다. 분모 하한 `max(s90, 0.5)`로 초소표본의 폭주를 막는다.
- **상한(cap)과 클램프**: 항별 상한(min(10, ...) 등)으로 한 지표의 이상치가 능력치를 지배하지 못하게 하고, 최종값은 30~96으로 클램프한다.

### 워크드 예제: 황인범 (MF, 3경기 263분, 1골 1도움)

| 능력치 | 계산 | 결과 |
|---|---|---|
| attack | 57 + min(10, 1.37×2.5)=3.43 + min(6, 0.68×4)=2.72 + 5 + 3 (w=1) = 71.1 | **71** |
| defense | 53 + min(12, (4/2.9)×5)=6.90 + min(8, (2/2.9)×4)=2.76 − 0 (w₂=1) = 62.7 | **63** |
| midfield | 73 + min(12, (2/2.9)×3)=2.07 + 4 + min(5,1)=1 + 2 = 82.1 | **82** |
| stamina | 85 − (29−27)×0.6=1.2 + 0 + MF 보정 4 = 87.8 | **88** |

산출 파일의 황인범 값(71/63/82/88)과 정확히 일치한다.

## 4. 검증 (Validation)

**(1) 재현성**: `python scripts/parse_stats.py` 재실행 후 `git diff data/squad_stats_2026.js` 가 **무변경(diff-zero)**. 커밋된 데이터가 스크립트만으로 완전 재생성됨을 확인했다 (2026-07-22 검증).

**(2) 분포 sanity (20인, 계산 능력치)**:

| 차원 | min | max | 평균 | 중앙값 |
|---|---|---|---|---|
| attack | 30 (GK) | 86 | 61.2 | 59.0 |
| defense | 36 | 90 | 59.2 | 56.5 |
| midfield | 40 (GK) | 86 | 65.2 | 65.0 |
| stamina | 80 | 90 | 86.3 | 87.0 |

전 값이 클램프 범위(30~96) 안에 있고, 포지션 프라이어 대비 상식적 산포를 보인다.

**(3) 서열 face-validity**: 공격 1위 오현규(86, 대회 유일 필드골 득점 공격수) > 손흥민(83), 수비 1위 김민재(90, 보정) > 이기혁(88, 팀내 최다 Int 5+TklW 4), 중원 1위 이강인(86, Crs 19) > 황인범(82, 1골 1도움). 대회 실적 서열과 정합적이다.

**(4) 커버리지 체크섬**: §1의 2,970분 검산.

## 5. 수동 보정과 한계 (Limitations)

- **명시적 보정 1건**: 김민재 `override_def: 90`. 3경기 표본의 Int 2 + TklW 2는 수비 라인 리더의 기여(포지셔닝, 커버)를 계수하지 못해 과소평가되므로 보정하고, 데이터 파일 주석에 `[수동 보정]`으로 남긴다. 보정은 이 1건뿐이다.
- **소표본**: 조규성(31분), 양현준(20분) 등은 per-90 수치가 불안정하다. 표본 가중 `w`로 억제하고 `statStr`에 "(소표본)"을 표기한다.
- **구조적 한계**: (a) 3경기 표본이라 상대 강도 보정이 없다. (b) Int/TklW는 수비 기여의 일부만 계수한다(압박, 커버 미계수). (c) 큐레이션 필드(rating 등)는 주관이 개입된다. 개선 방향: 예선·친선 포함 표본 확대, possession-adjusted 수비 지표, 큐레이션 필드의 공식화.

## 6. 엔진으로의 연결 (Consumption)

산출된 20인 능력치는 런타임에서 다음 경로로 소비된다. 전 구간이 로컬 연산($0)이다.

1. **팀 밸런스 HUD**: `updateStats()`가 필드 XI 평균 + 포메이션/다이얼 보정으로 4대 바를 실시간 재계산.
2. **λ 유도**: `secondHalfLambdas()`가 팀 능력치와 상대 전력으로 후반 기대득점 λ를 만들고, 전술 다이얼·체력·AI 상대감독 카운터가 곱셈 보정으로 붙는다.
3. **1,000회 몬테카를로**: Poisson(λ) 샘플링으로 승/무/패 분포를 집계.
4. **XAI 요인 분해**: 결과 카드가 λ의 기여 요인을 "승부 요인 TOP 3 + 상세 표"로 역전개해, 이 파이프라인의 데이터가 결과에 어떻게 닿았는지 유저에게 그대로 설명한다.
