# Model Selection & Validation — K-Tactics Lab 2026

이 문서는 경기 결과 모델의 **선택 근거**와 **사후 검증 결과**를 기록한다.

**프레이밍에 대한 정직성 선언**: 아래 §2의 대안 비교는 설계 시점에 요구사항과 문헌 근거로 내린 판단이지, 여러 후보 모델을 동일 데이터로 벤치마크한 실험이 아니다 (국가대표 3경기 표본으로는 그런 실험 자체가 성립하지 않는다). 대신 **채택한 모델이 주장하는 성질은 §3에서 실측으로 검증**하며, 검증은 `node scripts/validate_model.js`로 누구나 재현할 수 있다. LLM 레이어(Coach V, AI 상대감독)의 모델 선택은 [`AI_ENGINEERING.md`](AI_ENGINEERING.md) §2의 매트릭스를 따른다.

## 1. 요구사항과 채택 모델

경기 결과 엔진의 요구사항: **(a)** 브라우저 로컬 실행, 런타임 $0, `file://` 오프라인 동작 **(b)** 밀리초 단위 응답 **(c)** 승/무/패 확률과 **스코어 분포**까지 산출 (최빈 스코어 표시, 무승부 시 PK 분기 판정에 필요) **(d)** 결정론적 파이프라인으로 검증 가능 **(e)** 요인별로 분해 가능해 XAI 패널로 설명 가능.

**채택: 독립 Poisson 득점 모델 + 몬테카를로 집계.** 축구 득점을 Poisson 과정으로 근사하는 것은 Maher(1982)의 독립 Poisson 골 모델 이후 축구 통계의 표준 출발점이고, Dixon-Coles(1997)가 그 실용 보정 계보를 세웠다. 본 프로젝트는 이 계보의 최단순형을 쓴다: 팀 능력치 비율로 기본 λ(후반 기대득점)를 유도하고, 전술 다이얼·체력·AI 상대감독 카운터를 곱셈 보정으로 얹은 뒤, Poisson(λ)에서 1,000회 샘플링해 분포를 집계한다 (`app.js`의 `secondHalfLambdas` / `poissonSample`(Knuth 알고리즘) / `runMonteCarlo`).

## 2. 대안 비교 (설계 근거)

| 후보 | 기각 이유 |
|---|---|
| **LLM에게 결과 생성** | 비결정론적, 확률이 보정되지 않음, 콜당 비용·지연, 오프라인 불가, 설명 불가. "예측"은 LLM의 가치가 아니다 ([`AI_ENGINEERING.md`](AI_ENGINEERING.md) §0·§2와 동일 원칙). |
| **Elo / 로지스틱 승률 모델** | 승/무/패 확률만 주고 **스코어 분포를 주지 못한다**. 최빈 스코어 표시와 무승부→PK 분기가 요구사항이라 부적합. |
| **bivariate Poisson / Dixon-Coles ρ 보정** | 무승부 상관을 개선하지만 상관 파라미터 추정에 리그 규모 데이터가 필요하다. 3경기 표본으로는 추정 불가라 과적합만 남는다. 추정할 수 없는 파라미터는 넣지 않는 것이 정직하다. |
| **ML 회귀 (xG 기반 등)** | 학습 표본 부재 (동일 이유). 재현성·설명성도 하락. |

**해석적으로 풀 수 있는데 왜 몬테카를로인가**: 독립 Poisson의 승/무/패는 PMF 합산으로 닫힌 계산이 가능하다 (§3에서 실제로 그렇게 정답을 만든다). 그럼에도 샘플링을 쓰는 이유는 (a) 교체 이벤트·PK 등 비해석적 확장에 열려 있고 (b) "1,000번 돌린 시뮬레이션"이 유저에게 확률의 의미를 직관적으로 전달하며 (c) N=1,000이 밀리초에 끝나 비용이 없기 때문이다. 대신 샘플러가 해석해와 일치하는지를 검증으로 증명한다.

## 3. 사후 검증 (2026-07-22 실측, `scripts/validate_model.js`)

고정 보드 상태(기본 다이얼, vs 스페인, 전반 0:1, 체력 감소 반영)에서 λ_kor=0.719, λ_opp=0.770. 이 상태의 **해석적 정답**(PMF 합산, k≤20 절단)과 샘플러 결과를 비교했다.

**(1) 보정 (calibration): 샘플러는 불편(unbiased)이다**

| | 승 | 무 | 패 |
|---|---|---|---|
| 해석적 정답 | 8.90% | 21.14% | 69.96% |
| MC 200,000회 | 8.88% | 20.99% | 70.13% |
| MC 1,000회 (앱 설정, 1회 예시) | 8.1% | 21.2% | 70.7% |

**(2) 표본평균 = λ**: 후반 득점 표본평균(200k회)이 한국 0.716 vs λ 0.719, 상대 0.771 vs λ 0.770 (전반 1골 제외 후). Knuth 샘플러가 목표 분포를 정확히 따른다.

**(3) 수렴: N=1,000 선택 근거**. 반복수별 승률 추정치의 표준편차 (300회 반복 실측):

| N | 승률 추정 표준편차 | 비고 |
|---|---|---|
| 100 | ±2.8%p | UI 표시 단위(1%p) 대비 잡음 과다 |
| **1,000 (채택)** | **±0.9%p** | 표시 정밀도와 일치, 밀리초 내 완료 |
| 10,000 | ±0.3%p | 10배 연산으로 얻는 이득이 표시 단위 이하 |

이론 표준오차 √(p(1−p)/N) = √(0.089×0.911/1000) ≈ 0.90%p와 실측 ±0.92%p가 일치한다. 즉 N=1,000은 "적당히 큰 수"가 아니라 **표시 정밀도(정수 %)에 맞춘 통계적 선택**이다.

재현: `node scripts/validate_model.js` (앱의 실제 함수를 `app.js`에서 추출해 검사하므로 코드 복사·드리프트가 없다).

## 4. 한계와 정직한 경계선

- **독립 가정**: 양 팀 득점의 상관(무승부 쏠림)을 보정하지 않는다. §2의 이유로 의도된 단순화다.
- **다이얼 배수는 추정치가 아니라 설계값이다**: λ에 곱해지는 전술 보정(닥공 ×1.18 등)은 데이터에서 추정한 계수가 아니라 **게임 밸런스 설계 파라미터**다. 이를 숨기지 않고, 오히려 XAI 요인 분해 패널이 각 설계값의 효과를 유저에게 그대로 공개한다. 검증이 보증하는 것은 "주어진 λ에서 확률 계산이 정확하다"까지이며, "λ가 현실 경기를 정확히 예측한다"가 아니다.
- **상대 전력(`OPP_STRENGTH`)은 큐레이션**: 데이터 파이프라인([`DATA_PIPELINE.md`](DATA_PIPELINE.md))의 큐레이션 필드와 같은 지위로 명시한다.
